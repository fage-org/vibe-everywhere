## Highlights

- Reset the control app around the mobile-first AI worktree baseline with `首页 / 项目 / 通知 / 我的`
  top-level navigation and `会话 / 变更 / 文件 / 日志` project workspaces.
- Add execution-mode visibility and enforcement across conversation UI, relay persistence, and
  agent/provider runtimes, including safer writable-mode confirmation and provider-specific policy
  summaries.
- Upgrade project review and desktop worktree management with file-level Git diff loading,
  worktree create/remove actions, inventory retention, notification recall, and settings-side audit
  visibility.
- Add initial frontend `vitest` coverage for policy/runtime selection and project-inventory
  suppression semantics to reduce UI regression risk around worktree removal and policy display.

## Operator Notes

- Existing relay and agent deployment inputs stay compatible with `0.1.13`; this release focuses on
  product IA, conversation/review behavior, and frontend/runtime safety alignment rather than new
  deployment requirements.
- Non-current worktree removal now suppresses deleted worktrees from project inventory instead of
  leaving them visible as stale unreachable entries.
- Settings and notification preference surfaces now reflect real runtime/provider state more
  accurately, including ACP provider labeling and project-level override/inheritance copy.
