## Highlights

- Replaced the old session-monitoring dashboard with a conversation-first remote coding workflow:
  long-lived conversations are now the primary product surface, while Git, workspace, terminal,
  preview, and raw traces stay available as secondary inspectors.
- Added durable relay-backed conversation records plus provider-native continuation handles so
  follow-up turns continue the same remote coding thread instead of forcing users into temporary
  one-off sessions.
- Improved ACP correctness for remote coding providers: only OpenCode is advertised as standard
  ACP today, stable OpenCode ACP updates are mapped into the transcript correctly, and transcript-
  safe continuation now prefers standard `session/resume` when the agent advertises it.

## Included Iterations And Remediations

- Iteration v5 / Iteration 15 tranche: Conversation-First Threaded AI Sessions

## Operator Notes

- This release changes the primary operator-facing UX model from ad hoc AI sessions to durable
  conversation threads. Existing relay and agent deployment inputs are unchanged.
- Published CLI assets continue to use the existing operator-facing naming scheme with the tag in
  the filename, for example `vibe-everywhere-cli-v0.1.10-x86_64-unknown-linux-musl.tar.gz`.
- OpenCode remains the only provider advertised as standard ACP in this release. Codex and Claude
  Code continue to run through their current CLI-backed integration paths.
