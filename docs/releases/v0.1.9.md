## Highlights

- Switched the default Linux CLI release archive from a hosted-runner-coupled `gnu` build to a
  statically linked `x86_64-unknown-linux-musl` build so generic self-hosted Linux installs do not
  fail on older `glibc` baselines such as Debian 12.
- Updated Linux release packaging, installer asset resolution, and operator docs so the shipped
  Linux CLI target, download path, and deployment guidance all point to the same `musl` archive.
- Added explicit Linux CLI compatibility checks to `CI` and `Release`, including static-binary
  verification for the published `vibe-relay` and `vibe-agent` Linux artifacts.

## Included Iterations And Remediations

- Remediation v11: Linux CLI ABI Compatibility And musl Packaging

## Operator Notes

- The default Linux CLI release asset for this version is
  `vibe-everywhere-cli-v0.1.9-x86_64-unknown-linux-musl.tar.gz`.
- The Linux installer now resolves the `x86_64-unknown-linux-musl` archive by default on x86_64
  hosts.
- Windows packaging is unchanged for this release: `vibe-relay.exe` and `vibe-agent.exe` still
  need the packaged side-by-side EasyTier runtime files such as `Packet.dll`, `wintun.dll`, and
  `WinDivert64.sys`.
