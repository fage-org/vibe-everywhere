## Highlights

- Split relay auth into a control-plane token for human clients plus a separate agent enrollment
  token for first-time device registration.
- Agents now persist an issued device credential under `.vibe-agent/identity.json` and reuse that
  identity for heartbeats, polling, workspace/Git requests, and overlay bridge traffic.
- The relay no longer trusts external `x-vibe-*` actor headers, and device-only routes now enforce
  issued device credentials instead of inheriting control-plane access.

## Included Iterations And Remediations

- Remediation v10: Relay And Agent Auth Boundary Hardening

## Operator Notes

- Relay install scripts now accept both `RELAY_ACCESS_TOKEN` and `RELAY_ENROLLMENT_TOKEN`.
- Web, Desktop, and Android control clients should keep using `VIBE_RELAY_ACCESS_TOKEN`.
- Agents should use `VIBE_RELAY_ENROLLMENT_TOKEN` for initial enrollment and then reuse the issued
  device credential stored under `.vibe-agent/identity.json`.
